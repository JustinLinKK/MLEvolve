# mini_graph_demo

- Standalone demonstration of the graph database architecture defined under `schema/`.

- Seeds 17 nodes + 21 relationships into the running Neo4j instance, covers every node type and relationship type in `schema/graph_schema.yaml`, and ships three verification scripts plus an in-browser visualizer.

## Prerequisites

- Neo4j 5.x running at `bolt://127.0.0.1:7687`. In this repo the instance comes from the `neo4j-migrate` docker container.

- `export LOCALML_SCHEDULER_NEO4J_PASSWORD=<password>`. For the local docker instance the password is `test12345`.

- Repo-root venv with `neo4j` and `pyyaml` (already present at `.venv`).

## Usage

```bash
# Seed the demo subgraph (re-runnable, idempotent)
LOCALML_SCHEDULER_NEO4J_PASSWORD=test12345 .venv/bin/python -m mini_graph_demo.seed

# Pretty-print example queries
LOCALML_SCHEDULER_NEO4J_PASSWORD=test12345 .venv/bin/python -m mini_graph_demo.queries

# Run verifications (each exits non-zero on failure)
LOCALML_SCHEDULER_NEO4J_PASSWORD=test12345 .venv/bin/python -m mini_graph_demo.verify_constraints
LOCALML_SCHEDULER_NEO4J_PASSWORD=test12345 .venv/bin/python -m mini_graph_demo.verify_coverage
LOCALML_SCHEDULER_NEO4J_PASSWORD=test12345 .venv/bin/python -m mini_graph_demo.verify_queries

# Export the demo subgraph to Cytoscape JSON
LOCALML_SCHEDULER_NEO4J_PASSWORD=test12345 .venv/bin/python -m mini_graph_demo.export
```

## Visualize in browser

- Serve the `mini_graph_demo/` directory:

```bash
.venv/bin/python -m http.server 8766 --directory mini_graph_demo
```

- Open `http://localhost:8766/visualize.html` in any modern browser.

- The page renders 17 nodes (color-coded by label) and 21 edges (labeled by relationship type), with switchable layouts (dagre LR / TB, breadthfirst, concentric).

- Click any node to inspect its full property set in the lower panel.

## Isolation

- All primary keys are prefixed with `mgd:` and every node and relationship sets `demo_run = true`.

- The cleanup query in `seed.py` only deletes nodes with `demo_run = true`, so unrelated data in the local Neo4j is untouched.

## Files

- `fixtures.py` — pure-data dataclasses + `build_fixture()`

- `_driver.py` — Bolt connection helper (uses env var `LOCALML_SCHEDULER_NEO4J_PASSWORD`)

- `seed.py` / `queries.py` / `export.py` — write / read / dump

- `verify_constraints.py` / `verify_coverage.py` / `verify_queries.py` — three verifiers

- `tests/test_fixtures.py` — pytest unit tests for the pure-data parts

- `visualize.html` — Cytoscape.js in-browser visualizer

- `graph.json` — exported graph state (regenerated by `export.py`)
